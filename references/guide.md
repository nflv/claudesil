# Guide

## What this skill is for

This skill helps create a repository that is both:

1. a practical starter kit for Claude Code workflows
2. a teaching guide for how project memory and iterative correction should work

The repository should answer two questions clearly:

- How should I structure `CLAUDE.md` and related rules?
- How do repeated mistakes get converted into durable improvements?

## Explain the self-improvement loop precisely

Do not frame the loop as hidden model learning.
Frame it as system design.

The correct explanation is:

1. Claude produces an output.
2. A mistake or weak pattern is noticed.
3. The user identifies the root cause.
4. That cause is converted into a written guardrail.
5. Future sessions begin with better instructions.

This is why `CLAUDE.md` matters. It stores explicit, durable corrections.

## Teaching angle

A good repo should not only include files. It should show why the files exist.

For every major file, explain:

- its purpose
- what belongs there
- what does not belong there
- how it interacts with the rest of the system

## Design rule

The repo should be understandable by someone who has never used Claude Code before, but still useful to advanced users.
Balance beginner clarity with concrete operational detail.
