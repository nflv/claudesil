# Repository Map

## Root files

### README.md
The public entrypoint. Explain positioning, philosophy, installation, structure, and usage.

### CLAUDE.md
The highest-value operating file. Keep it concise. Put only non-obvious, reusable rules here.

### CLAUDE.local.example.md
A safe example for personal overrides that should not be committed with team defaults.

## .claude/

### .claude/rules/
Topic-specific rules that would otherwise bloat `CLAUDE.md`.

### .claude/workflows/
Multi-step procedures and decision trees.

## tasks/
Planning and review templates. Useful for plan mode and progress tracking.

## learnings/
Institutional memory. Every repeated failure should leave evidence here.

## diagrams/
Visuals that explain feedback loops and system architecture.

## examples/
Before/after content that proves the repository is practical, not theoretical.
