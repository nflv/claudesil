# Workflow

## Default flow for this skill

1. Read the user context and infer the repo's audience.
2. Start from the template repository in `assets/template-repo/`.
3. Adapt the template to the user's naming, conventions, and examples.
4. Ensure the repo contains both explanation and working files.
5. Check that each rule is concrete.
6. Check that each example demonstrates a learning loop.

## The loop to teach

Use this model throughout the repo:

- output
- failure
- diagnosis
- written rule
- improved future run

## Minimal operating doctrine

- simplicity first
- no lazy fixes
- minimal blast radius
- verify before done

## Adoption sequence for new users

1. Start with the base `CLAUDE.md`.
2. Add only 10 to 20 lines that matter.
3. Add modular rules only after real mistakes appear.
4. Store examples of corrected failures.
5. Review and prune instructions regularly.
